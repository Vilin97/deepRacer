Findings:

  1. Need to penalize crashing much more than going fast.
  2. Simple reward functions seem to work best,
  3. Training on the oval track generalizes well to other (simple) tracks.
  4. Training for 30 minutes is enough to get an idea of how good the reward function is
  5. Depending on the reward function at the edge of the track, the agent will “suicide” it feels things are hopeless.
  1. What’s happening here, I think, is the agent is basically trying to minimize negative rewards, and the sure-fire way to do that is to end the simulation.
  1. I notice that PPO’s training stats all look pretty stable, whereas SAC’s oscillate crazy. PPO provides much better stability of training. See models https://us-east-1.console.aws.amazon.com/deepracer/home?region=us-east-1#model/0023d37a-4a76-4802-9517-249806e4b47f and https://us-east-1.console.aws.amazon.com/deepracer/home?region=us-east-1#model/00ea5f9e-b6e5-4ad2-ab85-a84832bbb225 are basically identical, with the main difference being PPO vs SAC. The difference in graphs is stark.
  2. Short-term reward don’t work well – the agent over-optimizes for them, losing sight of the long-term goal

Ideas to try:

  1. Penalize steering too much to prevent zig-zagging.
  2. Negatively reward for crashing, not for having a wheel off-track.
  3. At each point, only change one parameter of a successful model.
  4. Decrease discount to 0.995 or 0.997
  5. Use steps to make the agent finish faster
  6. Train another identical model to see how big of a role randomness plays in training – Not that big, see duplicate-first-try.
  7. Train an agent with max speed 2 for 2 hours, and another agent building up to max speed 2.
  8. I wonder if we should experiment with the discount factor. 0.997^300 = 0.4, 0.999^300 = 0.74. This is a huge difference (and for longer tracks with more than 300 steps -- even larger). Maybe I should train 3 agents -- with discounts 0.999, 0.997, 0.995 -- and identical otherwise. Say, for an hour with max speed 1.2 or something like that.

Lessons from blog posts:

  1. - increase the minimum speed to about 1. Limit max speed for high turning angles.
  2. - give very high reward for finishing the race (progress = 100)
  3. - keep reward between -1 and 1. They actually keep it between 0 and 1, giving 0 for off-track
  4. - split the reward function into multiple components: short term and long term rewards. Short term would be things like speed and whether we're facing towards the next waypoint. Long term would be making progress in few steps.
  5. - they don't just reward progress but fast progress, using steps.

Computing radius of circle:

# Input 3 coords [[x1,y1],[x2,y2],[x3,y3]]

def circle_radius(coords):

    # Flatten the list and assign to variables

    x1, y1, x2, y2, x3, y3 = [i for sub in coords for i in sub]

    a = x1*(y2-y3) - y1*(x2-x3) + x2*y3 - x3*y2

    b = (x1**2+y1**2)*(y3-y2) + (x2**2+y2**2)*(y1-y3) + (x3**2+y3**2)*(y2-y1)

    c = (x1**2+y1**2)*(x2-x3) + (x2**2+y2**2)*(x3-x1) + (x3**2+y3**2)*(x1-x2)

    d = (x1**2+y1**2)*(x3*y2-x2*y3) + (x2**2+y2**2) * \

        (x1*y3-x3*y1) + (x3**2+y3**2)*(x2*y1-x1*y2)

    # In case a is zero (so radius is infinity)

    try:

        r = abs((b**2+c**2-4*a*d) / abs(4*a**2)) ** 0.5

    except:

        r = 999

    return r

where alpha is the steering angle, r is the radius of fitted circle, and 𝐿=0.165𝑚 is the axle length.
